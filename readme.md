# Peer Sharing WireGuard

Simple infrastructure for running a **WireGuard VPN server using Docker**.

The goal of this repository is to provide a **minimal and reproducible setup** for creating a VPN server and generating peer configurations.

This repo only contains **infrastructure and templates**.
All **private keys and runtime configs are generated locally** and are ignored by Git.

---

# Project Structure

```
peer-sharing-wireguard
│
├ docker-compose.yml       # Main docker orchestration
│
├ templates/               # WireGuard configuration templates
│  ├ server.conf
│  └ peer.conf
│
├ coredns/                 # Internal DNS configuration
│  └ Corefile
│
├ wireguard-config/        # Runtime generated configs (ignored by git)
│
└ .gitignore
```

`wireguard-config/` is created automatically and contains:

```
wireguard-config/
 ├ server/
 ├ peer1/
 └ wg_confs/
```

These include:

* private keys
* peer configs
* QR codes
* runtime VPN configs

These files **must never be committed to Git**.

---

# Requirements

You need the following installed:

* Docker
* Docker Compose

Check installation:

```
docker --version
docker compose version
```

---

# Running the Server

Start the infrastructure:

```
docker compose up -d
```

This will:

1. Start the WireGuard container
2. Generate configuration files
3. Create the `wireguard-config` directory
4. Bring up the VPN interface

---

# Stopping the Server

```
docker compose down
```

---

# Generated Files

After running the system, the following files appear:

```
wireguard-config/
```

This directory contains:

* server keys
* peer keys
* peer configuration files
* QR codes for mobile clients

These are runtime artifacts.

---

# Adding a Peer

Peers are generated by creating a new peer configuration and restarting the container.

Typical output for a peer:

```
peer1.conf
peer1.png
```

The QR code can be scanned directly in the WireGuard mobile app.

---

# Security Notes

This repository intentionally **does not store any secrets**.

Never commit:

* private keys
* preshared keys
* peer configs
* QR codes

These files are automatically ignored by `.gitignore`.

---

# Purpose of This Repo

This project exists mainly for:

* learning WireGuard infrastructure
* quick VPN deployment
* personal peer network experiments
* Docker-based reproducible setup

---

# Notes for Future Me

If something breaks:

1. Delete the runtime folder:

```
rm -rf wireguard-config
```

2. Restart the system:

```
docker compose up
```

This regenerates everything from scratch.
